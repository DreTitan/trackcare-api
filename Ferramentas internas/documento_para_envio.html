HealthGoClient

Desktop Application - Suporte
Explicacao

Aplicativo desktop utilizado pelos especialistas de Suporte. Vinculado ao servidor interno da HealthGo com conexao ao banco de dados Nocobase/AWS e Datalake para envio de SVM ao HealthGo Studio.

Requisitos Tecnicos
Categoria Requisito
Sistema Operacional Windows 10 ou superior
Rede Netbird VPN conectado ao SV HealthGo
Processador Intel Core i5 ou superior
Memoria RAM 8GB
Disco 256GB SSD
Maiores Gargalos
  • Alta
    Instabilidade por Queda do Servidor
    Quando o servidor principal cai, o HealthGoClient tambem fica indisponivel
  • Alta
    Dependencia do Datalake para Upload de HGID
    Se a Producao nao subir o HGID para o Datalake, nao e possivel emitir SVM
  • Media
    Sincronizacao com HealthGoStudio
    HGIDs sincronizados no Client podem nao aparecer imediatamente no Studio ate liberacao manual
IMPACTO NA DEVOLUTIVA AO CLIENTE

Problemas no Client afetam diretamente o atendimento na clinica:

  • Interrupcao do atendimento quando servidor cai
  • Bloqueio na emissao de SVMs quando HGID nao esta no Datalake
  • Atrasos por liberacao manual entre Client e Studio
  • Impacto direto na experiencia do cliente final

HealthGoStudio

Ferramenta Interna - Suporte N2/N3
Explicacao

Ferramenta de Suporte para uso operacional do N2 e N3. Vinculado ao servidor interno da HealthGo com conexao ao banco de dados Nocobase/AWS para armazenamento de todos os dados dos equipamentos bem como, sensores, componentes, placas, firmware, software e demais dados.

Funcao Principal
  • Gestao de SVM Subir SVM do Datalake para Nocobase e emitir para instalacao no cliente
  • Importar Dados de Calibracao Processar dados de calibracao dos sensores
  • Processamento de Dados Remocao de outliers, analise estatistica
  • Modelo SVM Treinar novo modelo SVM, revisar predicoes e graficos (N3 e Engenharia)
  • Compilacao de Firmware Compilar firmware do dispositivo selecionado e enviar para a nuvem (Nocobase/AWS)
Requisitos Tecnicos
Categoria Requisito
Sistema Operacional Windows 10 ou superior
Rede Netbird VPN conectado ao SV HealthGo
Processador Intel Core i5 ou superior
Memoria RAM 8GB
Disco 256GB SSD
Maiores Gargalos
  • Alta
    Dashboard em Tempo Real com Dados Imprecisos
    A dashboard em tempo real envia dados com baixa precisao e velocidade, frequentemente fica offline
  • Media
    Falta de Visibilidade de Manutencao Preventiva
    Nao e possivel visualizar o numero de exames realizados por HGID (250 exames ou 1 ano)
  • Media
    Falta de Controle Remoto de Componentes
    Nao ha funcionalidade de bypass remoto para componentes do equipamento
IMPACTO NA DEVOLUTIVA AO CLIENTE

Dados imprecisos em tempo real dificultam diagnostico. Analise precisa depende da estabilidade da dashboard.

  • Dados imprecisos em tempo real dificultam diagnostico
  • Restricoes de firmware atrasam testes de solucao
  • Falta de contador de exames impede previsao de manutencoes
  • Problemas de hardware nao sao resolvidos remotamente
